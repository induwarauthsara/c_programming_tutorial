üìã Question 3: Pointers (Total: 25 marks)

Question 3(a) - What is a Pointer? [3 marks]

Question: What is a pointer in C? Explain using a suitable example.

üîç Click to See Complete Answer
Complete Answer:

Definition: A pointer in C is a variable that stores the memory address of another variable. Instead of holding a data value directly, it holds the location where a data value is stored.

Key Uses:

  • Dynamic memory allocation
  • Working with arrays efficiently
  • Implementing call-by-reference in functions

Key Operators:

  • & (address-of): Gets the memory address of a variable
  • * (dereference): Accesses the value stored at an address
#include <stdio.h> int main() { int value = 100; // Declare an integer variable 'value' int *ptr; // Declare a pointer variable 'ptr' ptr = &value; // Assign the memory address of 'value' to 'ptr' printf("Value of 'value': %d\n", value); // Output: 100 printf("Address of 'value': %p\n", &value); // Output: (e.g., 0x7ffeee00b000) printf("Value of 'ptr' (address it holds): %p\n", ptr); // Output: (same as address of 'value') printf("Value pointed to by 'ptr': %d\n", *ptr); // Output: 100 *ptr = 200; // Change the value at the address 'ptr' points to printf("New value of 'value' after pointer modification: %d\n", value); // Output: 200 return 0; }
Step-by-Step Explanation:
  1. Declaration: int *ptr; declares ptr as a pointer to an integer
  2. Assignment: ptr = &value; makes ptr point to value
  3. Access: *ptr accesses the value that ptr points to
  4. Modification: *ptr = 200; changes the original variable through the pointer

Question 3(b) - Pointer Concepts True/False [14 marks]

Instructions: For each code snippet, determine if the statements are True (T) or False (F).

Part (i) - Pointer to Const
int a = 10; const int b = 30; const int *p2 = &b; p2 = &a; printf("%d\n", *p2);
Statements to Evaluate:
  1. p2 points to a constant.
  2. *p2 = 40; is a valid assignment that modifies the value of variable b.
  3. p2 = &a; is a valid assignment as p2 can points to another variable.
  4. The output of the printf statement is 20.
üìã Click for Answers & Explanations
1. True (T)

const int *p2 means p2 is a pointer to a constant integer. The value pointed to cannot be changed through p2.

2. False (F)

*p2 = 40; is invalid because p2 points to a const int. Even if allowed, after p2 = &a;, *p2 would refer to a, not b.

3. True (T)

p2 itself is not a constant pointer (only the data it points to is constant). Therefore, p2 can be reassigned to point to different variables.

4. False (F)

After p2 = &a;, p2 points to a, whose value is 10. The printf statement would output 10, not 20.

Part (ii) - Const Pointer
int a = 10; int c = 50; int *const p3 = &c; *p3 = 60; printf("%d\n", c);
Statements to Evaluate:
  1. p3 is a pointer to constant which will allow you to change the value of the variable c.
  2. The statement *p3 = 60; allows modifying the value of c through p3.
  3. As p3 is a constant pointer, if you want it can be reassigned as p3 = &a;
  4. The output of printf statement is 60.
üìã Click for Answers & Explanations
1. False (F)

int *const p3 means p3 is a constant pointer to an integer. The pointer itself cannot be changed, but the value it points to can be modified.

2. True (T)

*p3 = 60; is valid because p3 points to a non-constant integer (c). Only p3's address cannot change, not the data it points to.

3. False (F)

Since p3 is a const pointer, its value (the address it holds) cannot be changed. p3 = &a; is an invalid reassignment.

4. True (T)

c was 50, then *p3 = 60; changed c to 60. The printf statement will output 60.

Part (iii) - Regular Pointer
int a = 10; int *p1 = &a; *p1 = 20; printf("Value of a = %d\n", a);
Statements to Evaluate:
  1. Modifying the value of a through p1 is not possible.
  2. Output of the program code is 10.
üìã Click for Answers & Explanations
1. False (F)

int *p1 = &a; declares a regular pointer p1 to an integer. *p1 = 20; correctly modifies the value of a through the pointer p1.

2. False (F)

The variable a was initialized to 10, then modified to 20 by *p1 = 20;. The printf statement will output "Value of a = 20".

Part (iv) - Const Pointer to Const
int a = 10; const int d = 70; const int *const p4 = &d; //printf("%d\n", *p4);
Statements to Evaluate:
  1. p4 is a constant pointer to a constant variable.
  2. The statement *p4 = 80; will not be allowed in this context.
  3. p4 is a constant pointer, it can't be reassigned as p4 = &a;
  4. The output of printf statement is the memory address of variable d.
üìã Click for Answers & Explanations
1. True (T)

const int *const p4 declares p4 as a constant pointer to a constant integer. Neither the pointer nor the value it points to can be changed.

2. True (T)

*p4 = 80; is invalid because p4 points to a constant integer. The value d cannot be modified via p4.

3. True (T)

As p4 is a const pointer, its memory address cannot be changed. p4 = &a; is an invalid assignment.

4. False (F)

If the printf were uncommented, it would print the value 70 (value of d), not its memory address. Use %p for addresses.

üéì Exam Tips & Key Takeaways

For Pointer Questions:
  • Remember the const variations:
    • const int *p - pointer to constant (can't change value)
    • int * const p - constant pointer (can't change address)

Your First Pointer Program

Let's create our first pointer step by step:

  1. Create a regular variable to point to
  2. Create a pointer variable to store the address
  3. Make the pointer point to our variable
  4. Use the pointer to access the value
#include <stdio.h> int main() { // Step 1: Create a regular variable int number = 42; // Step 2: Create a pointer variable // int* means "pointer to int" int *pointer; // Step 3: Make pointer point to number // &number means "address of number" pointer = &number; // Step 4: Use the pointer printf("Direct access: %d\n", number); printf("Through pointer: %d\n", *pointer); return 0; }

Breaking Down Each Line:

  • int *pointer; - Creates a pointer that can point to integers
  • &number - Gets the memory address of 'number'
  • *pointer - Goes to the address and gets the value stored there

Key Symbols:

  • * (in declaration) = "pointer to"
  • & = "address of"
  • * (when using) = "value at address"

Practical Example: Swapping Two Numbers

üßê The Problem

Imagine you want to write a function that swaps two numbers. Without pointers, this is impossible because function parameters are just copies!

// This WON'T work - why? void wrong_swap(int a, int b) { int temp = a; a = b; b = temp; // Only swaps the copies! } // This WILL work - using pointers! void correct_swap(int *a, int *b) { int temp = *a; // Get value at address a *a = *b; // Put value from b into a *b = temp; // Put temp into b } int main() { int x = 10, y = 20; printf("Before: x=%d, y=%d\n", x, y); // Pass addresses, not values correct_swap(&x, &y); printf("After: x=%d, y=%d\n", x, y); return 0; }

Why Pointers Work Here:

  1. We pass addresses (&x, &y) instead of values
  2. Function receives addresses in pointer parameters
  3. We modify values at those addresses using *
  4. Original variables change because we modified their memory locations

Expected Output:

Before: x=10, y=20
After: x=20, y=10

Understanding Const with Pointers

üö® This is Confusing for Beginners!

The combination of const and pointers can be tricky. Let's break it down with simple rules.

üìç Pointer to Const

const int *ptr

Rule: Can't change the VALUE through this pointer

Allowed: Point to different variables

Analogy: Read-only access to different books

üîí Const Pointer

int * const ptr

Rule: Can't change WHERE the pointer points

Allowed: Change the value at that location

Analogy: Permanent address, can redecorate the house

üîê Const Pointer to Const

const int * const ptr

Rule: Can't change ANYTHING

Analogy: Museum display - look but don't touch

#include <stdio.h> int main() { int a = 10, b = 20; // 1. Pointer to const - can't change value const int *ptr1 = &a; printf("Value: %d\n", *ptr1); // OK - reading // *ptr1 = 30; // ERROR! Can't change value ptr1 = &b; // OK - can point elsewhere // 2. Const pointer - can't change address int * const ptr2 = &a; *ptr2 = 30; // OK - can change value // ptr2 = &b; // ERROR! Can't point elsewhere // 3. Const pointer to const - can't change anything const int * const ptr3 = &a; printf("Value: %d\n", *ptr3); // OK - reading only // *ptr3 = 40; // ERROR! Can't change value // ptr3 = &b; // ERROR! Can't change address return 0; }

Pointers and Arrays - Best Friends!

üè† Arrays as Apartment Buildings

Think of an array as an apartment building:

  • Each apartment has a number (index 0, 1, 2, 3...)
  • The building has a street address (array name points to first apartment)
  • You can walk from one apartment to the next (pointer arithmetic)
#include <stdio.h> int main() { int numbers[] = {10, 20, 30, 40, 50}; int *ptr = numbers; // Same as &numbers[0] printf("Array access methods:\n"); // Method 1: Traditional array indexing for(int i = 0; i < 5; i++) { printf("numbers[%d] = %d\n", i, numbers[i]); } printf("\nPointer access methods:\n"); // Method 2: Pointer with indexing for(int i = 0; i < 5; i++) { printf("*(ptr + %d) = %d\n", i, *(ptr + i)); } printf("\nMoving through array with pointer:\n"); // Method 3: Moving the pointer for(int i = 0; i < 5; i++) { printf("*ptr = %d (address: %p)\n", *ptr, (void*)ptr); ptr++; // Move to next element } return 0; }

Common Beginner Mistakes

‚ö†Ô∏è Mistake 1: Forgetting the *

int x = 10; int *ptr = &x; // WRONG: printf("%d\n", ptr); // Prints address, not value! // CORRECT: printf("%d\n", *ptr); // Prints 10

‚ö†Ô∏è Mistake 2: Uninitialized Pointers

int *ptr; // DANGEROUS! Points to random memory // *ptr = 10; // Could crash your program! // CORRECT: int x = 5; int *ptr = &x; // Now it's safe to use *ptr = 10; // OK!

üîß Debugging Pointer Problems

  1. Always initialize pointers before using them
  2. Check if pointer is NULL before dereferencing
  3. Use %p to print addresses when debugging
  4. Remember: ptr gives address, *ptr gives value

üèóÔ∏è Structures - Building Complex Data Types

ü§î "Why do I need structures?"

Imagine you're organizing information about students. You could use separate variables for each piece of data, but that gets messy quickly. Structures let you bundle related information together!

The Problem Structures Solve

üìã The Student Information Problem

Without structures, managing student data is chaotic:

// Messy way - separate variables for each student int student1_id = 101; char student1_name[50] = "Alice"; float student1_gpa = 3.8; int student2_id = 102; char student2_name[50] = "Bob"; float student2_gpa = 3.5; // This gets unmanageable very quickly!

With structures, it's organized and clean:

struct Student { int id; char name[50]; float gpa; }; struct Student student1 = {101, "Alice", 3.8}; struct Student student2 = {102, "Bob", 3.5};

Your First Structure

Creating a structure involves these steps:

  1. Define the structure template - What data will it hold?
  2. Create structure variables - Actual instances of the structure
  3. Initialize the data - Put values into the structure
  4. Access the data - Use dot notation to get/set values
#include <stdio.h> #include <string.h> // Step 1: Define the structure template struct Person { char name[50]; // Member 1 int age; // Member 2 float height; // Member 3 char city[30]; // Member 4 }; int main() { // Step 2: Create structure variables struct Person person1; struct Person person2; // Step 3: Initialize person1 strcpy(person1.name, "John Doe"); person1.age = 25; person1.height = 5.9; strcpy(person1.city, "New York"); // Alternative initialization for person2 struct Person person2_init = { "Jane Smith", 23, 5.6, "Los Angeles" }; person2 = person2_init; // Step 4: Access and display the data printf("Person 1 Information:\n"); printf("Name: %s\n", person1.name); printf("Age: %d years\n", person1.age); printf("Height: %.1f feet\n", person1.height); printf("City: %s\n\n", person1.city); printf("Person 2 Information:\n"); printf("Name: %s\n", person2.name); printf("Age: %d years\n", person2.age); printf("Height: %.1f feet\n", person2.height); printf("City: %s\n", person2.city); return 0; }

Understanding Each Part:

  • struct Person { ... }; - Defines the template (blueprint)
  • struct Person person1; - Creates an actual variable
  • person1.name - Accesses the name member using dot notation
  • strcpy() - Copies strings into character arrays

Important: The structure definition doesn't create any variables - it's just a template!

Working with Arrays of Structures

üí° Why Arrays of Structures?

When you need to store information about multiple similar items (like a class of students, a team of employees, etc.), arrays of structures are perfect!

Pointers to Structures

üîó Why Use Pointers with Structures?

  • Efficiency: Avoid copying large structures
  • Modification: Allow functions to modify the original structure
  • Memory: Save memory when passing to functions

üìã The File Cabinet Analogy

Think of a structure as a file cabinet:

  • Regular access: cabinet.drawer1 - You walk to the cabinet and open drawer 1
  • Pointer access: cabinet_ptr->drawer1 - You have the cabinet's address and ask someone to open drawer 1

Both get the same result, but the pointer method is more efficient for large cabinets!

#include <stdio.h> #include <string.h> struct Book { char title[50]; char author[30]; int pages; float price; }; // Function that takes structure pointer void displayBook(struct Book *book_ptr) { printf("Book Information:\n"); printf("Title: %s\n", book_ptr->title); printf("Author: %s\n", book_ptr->author); printf("Pages: %d\n", book_ptr->pages); printf("Price: $%.2f\n\n", book_ptr->price); } // Function that modifies structure through pointer void applyDiscount(struct Book *book_ptr, float discount) { book_ptr->price *= (1.0 - discount); printf("Applied %.0f%% discount to %s\n", discount * 100, book_ptr->title); } int main() { // Create and initialize book struct Book myBook; strcpy(myBook.title, "C Programming Guide"); strcpy(myBook.author, "John Smith"); myBook.pages = 450; myBook.price = 29.99; // Display original book info displayBook(&myBook); // Apply 20% discount applyDiscount(&myBook, 0.20); // Display updated book info displayBook(&myBook); return 0; }

Key Points About Structure Pointers:

  • -> operator: Used to access members through a pointer
  • (*ptr).member is the same as ptr->member
  • &myBook passes the address of the structure
  • Changes made through pointer affect the original structure

üîÑ Unions - Sharing Memory Efficiently

ü§î "What's the difference between structures and unions?"

Great question! While structures give each member its own memory space, unions make all members share the same memory space. Think of it like this:

  • Structure = Apartment building: Each resident has their own room
  • Union = Hotel room: Only one guest can use the room at a time

Understanding Memory Sharing

Memory Comparison: Structure vs Union

Structure Memory (Separate Spaces)
int num (4 bytes)
float decimal (4 bytes)
char letter (1 byte)
Total: 9 bytes (+ padding)
Union Memory (Shared Space)
Same 4 bytes used by:
‚Ä¢ int num
‚Ä¢ float decimal
‚Ä¢ char letter
Total: 4 bytes only!

Your First Union

#include <stdio.h> // Define a union - similar syntax to struct union SimpleData { int number; // 4 bytes float decimal; // 4 bytes char letter; // 1 byte }; int main() { union SimpleData data; printf("Size of union: %lu bytes\n", sizeof(data)); printf("Address of number: %p\n", (void*)&data.number); printf("Address of decimal: %p\n", (void*)&data.decimal); printf("Address of letter: %p\n\n", (void*)&data.letter); // Store an integer data.number = 1000; printf("Stored integer 1000:\n"); printf(" data.number = %d\n", data.number); printf(" data.decimal = %f (garbage!)\n", data.decimal); printf(" data.letter = %c (garbage!)\n\n", data.letter); // Store a float - overwrites the integer! data.decimal = 3.14159; printf("Stored float 3.14159:\n"); printf(" data.number = %d (garbage!)\n", data.number); printf(" data.decimal = %f\n", data.decimal); printf(" data.letter = %c (garbage!)\n\n", data.letter); // Store a character - overwrites the float! data.letter = 'A'; printf("Stored character 'A':\n"); printf(" data.number = %d (ASCII value of 'A')\n", data.number); printf(" data.decimal = %f (garbage!)\n", data.decimal); printf(" data.letter = %c\n", data.letter); return 0; }

Key Observations:

  • Same address: All members have the same memory address
  • Size of largest: Union size equals the largest member (4 bytes for int/float)
  • One at a time: Only one member can hold meaningful data
  • Overwriting: Storing in one member overwrites others

‚ö†Ô∏è Important Union Rules

  1. Only initialize the first member during declaration
  2. Only one member is valid at any given time
  3. Reading invalid members gives unpredictable results
  4. Size is determined by the largest member

üìÅ File Handling - Saving and Loading Data

ü§î "Why do I need to work with files?"

Think about it - when you close a program, all the data in variables disappears! Files let you:

  • Save data permanently - So it survives when the program ends
  • Share data - Between different programs or users
  • Process large amounts of data - That won't fit in memory all at once

It's like the difference between writing on a whiteboard (temporary) vs writing in a book (permanent)!

File Operations Overview

üìö Files are Like Books in a Library

  • fopen(): Taking a book off the shelf and opening it
  • fread(): Reading pages from the book
  • fwrite(): Writing new content in the book
  • fseek(): Jumping to a specific page number
  • ftell(): Checking which page you're currently on
  • fclose(): Closing the book and putting it back

üìñ fread()

Purpose: Read data from a file into memory

When to use: Loading saved data, processing file contents

Returns: Number of items successfully read

üìù fwrite()

Purpose: Write data from memory to a file

When to use: Saving data permanently, creating files

Returns: Number of items successfully written

üéØ fseek()

Purpose: Move to a specific position in the file

When to use: Random access, jumping to specific records

Options: SEEK_SET, SEEK_CUR, SEEK_END

üìç ftell()

Purpose: Get current position in the file

When to use: Finding file size, bookmarking positions

Returns: Current position as number of bytes

Understanding File Modes

File Opening Modes Explained:

  1. "r": Read only - File must exist
  2. "w": Write only - Creates new file or overwrites existing
  3. "a": Append - Adds to end of file
  4. "r+": Read and write - File must exist
  5. "w+": Read and write - Creates new file or overwrites
  6. "rb", "wb": Binary modes - For non-text data

Your First File Program - Writing Data

#include <stdio.h> #include <stdlib.h> struct Student { int id; char name[30]; float gpa; int age; }; int main() { FILE *file; struct Student students[] = { {101, "Alice Johnson", 3.8, 20}, {102, "Bob Smith", 3.5, 21}, {103, "Carol Davis", 3.9, 19}, {104, "David Wilson", 3.2, 22} }; // Step 1: Open file for writing in binary mode file = fopen("students.dat", "wb"); if (file == NULL) { printf("Error: Cannot create file!\n"); return 1; } // Step 2: Write all students to file size_t written = fwrite(students, sizeof(struct Student), 4, file); // Step 3: Check if write was successful if (written == 4) { printf("Successfully wrote %zu students to file!\n", written); printf("File size: %zu bytes\n", written * sizeof(struct Student)); } else { printf("Error: Only wrote %zu out of 4 students!\n", written); } // Step 4: Close the file fclose(file); printf("File closed successfully.\n"); return 0; }

Breaking Down fwrite():

fwrite(data, size_of_each_item, number_of_items, file_pointer)

  • students: Pointer to data to write
  • sizeof(struct Student): Size of each student record
  • 4: Number of students to write
  • file: File pointer from fopen

Return value: Number of items actually written (should equal number requested)

Expected Output:

Successfully wrote 4 students to file!
File size: 168 bytes
File closed successfully.

File Position Control with fseek() and ftell()

üéØ Why Control File Position?

Sometimes you don't want to read a file from beginning to end. Maybe you want to:

  • Jump to a specific record in a database file
  • Read the file backwards from end to beginning
  • Skip unwanted data and jump to interesting parts
  • Update specific records without rewriting the whole file
#include <stdio.h> #include <stdlib.h> int main() { FILE *file; char message[] = "Hello, World! This is a test file for seeking operations."; char buffer[20]; // Step 1: Create a test file file = fopen("test.txt", "w"); if (file != NULL) { fprintf(file, "%s", message); fclose(file); printf("Created test file with: \"%s\"\n\n", message); } // Step 2: Open file for reading and demonstrate seeking file = fopen("test.txt", "r"); if (file == NULL) { printf("Error opening file!\n"); return 1; } printf("File Position Demonstrations:\n"); printf("============================\n"); // Get initial position printf("1. Initial position: %ld\n", ftell(file)); // Read first 7 characters fread(buffer, 1, 7, file); buffer[7] = '\0'; printf("2. Read 7 chars: \"%s\"\n", buffer); printf("3. Position after reading: %ld\n\n", ftell(file)); // Seek to position 14 (start of "This") fseek(file, 14, SEEK_SET); printf("4. After seeking to position 14: %ld\n", ftell(file)); fread(buffer, 1, 4, file); buffer[4] = '\0'; printf("5. Read 4 chars from position 14: \"%s\"\n\n", buffer); // Seek relative to current position fseek(file, 3, SEEK_CUR); // Skip " is" printf("6. After seeking +3 from current: %ld\n", ftell(file)); fread(buffer, 1, 2, file); buffer[2] = '\0'; printf("7. Read 2 chars: \"%s\"\n\n", buffer); // Find file size by seeking to end fseek(file, 0, SEEK_END); long file_size = ftell(file); printf("8. File size: %ld bytes\n", file_size); // Read last 10 characters fseek(file, -10, SEEK_END); // 10 chars from end printf("9. Position 10 chars from end: %ld\n", ftell(file)); fread(buffer, 1, 10, file); buffer[10] = '\0'; printf("10. Last 10 characters: \"%s\"\n", buffer); fclose(file); return 0; }

Understanding fseek() Parameters:

fseek(file, offset, whence)

  • SEEK_SET: Move relative to beginning of file
  • SEEK_CUR: Move relative to current position
  • SEEK_END: Move relative to end of file

Offset: Positive = forward, Negative = backward

File Operations Best Practices

üõ°Ô∏è Essential File Handling Rules

  1. Always check if fopen() succeeds - Files might not exist or be locked
  2. Always close files - Use fclose() to free resources
  3. Check return values - fread/fwrite return actual items processed
  4. Use appropriate modes - Binary for structures, text for strings
  5. Handle errors gracefully - Don't crash when files are missing

üîß Common File Handling Problems

Problem 1: File not found
// Always check if file opened successfully FILE *file = fopen("data.txt", "r"); if (file == NULL) { printf("Error: Cannot open file!\n"); return 1; // Exit with error code }
Problem 2: Partial reads/writes
// Check if all data was processed size_t written = fwrite(data, sizeof(item), count, file); if (written != count) { printf("Warning: Only wrote %zu out of %d items\n", written, count); }
Problem 3: Forgetting to close files
// Always close files, even on error FILE *file = fopen("data.txt", "w"); if (file == NULL) return 1; // ... do file operations ... fclose(file); // Don't forget this!

Level 1: Basic Concepts (Beginners)

Exercise 1: Pointer Basics

Goal: Master basic pointer operations

Task:

  4. Use the pointer to change the value to 200
  5. Create a second pointer that points to the first pointer
Hint & Expected Output

Address of variable: 0x...

Exercise 2: Structure Practice

Goal: Create and manipulate structures

Task:

  1. Create a "Car" structure with: brand, model, year, price
  2. Create an array of 3 cars
  3. Write a function to display car information
  4. Write a function to find the most expensive car
  5. Write a function to apply a discount to all cars

Debugging and Testing Challenges

üêõ Bug Hunt Exercises

Each of these code snippets has bugs. Can you find and fix them?

Bug Hunt 1: Pointer Problems
int main() { int *ptr; *ptr = 100; // BUG: What's wrong here? printf("%d\n", ptr); // BUG: Another problem! return 0; }
Bug Hunt 2: Structure Issues
struct Person { char name[20]; int age; }; void updateAge(struct Person p, int newAge) { p.age = newAge; // BUG: Why won't this work? }
Bug Hunt 3: File Handling Errors
int main() { FILE *file = fopen("data.txt", "r"); char buffer[100]; fgets(buffer, 100, file); // BUG: What if file is NULL? printf("%s", buffer); // BUG: Missing something important! return 0; }
Click for Bug Solutions

Bug Hunt 1 Solutions:

  • Line 2: ptr is uninitialized - should point to a valid variable first
  • Line 3: Printing ptr (address) instead of *ptr (value)

Bug Hunt 2 Solution:

  • Parameter is passed by value - changes don't affect original. Use pointer parameter.

Bug Hunt 3 Solutions:

  • Line 3: No NULL check after fopen
  • Missing fclose() call
